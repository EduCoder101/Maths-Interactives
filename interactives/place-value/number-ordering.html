Number Ordering Detective

Master ascending and descending order

Learning Intention & Success Criteria ▼

Learning Intention

We are learning to compare and order whole numbers and decimals by reading them from left to right and comparing digits in each place value column.

Success Criteria

  • I can arrange numbers in ascending order (smallest to largest)
  • I can arrange numbers in descending order (largest to smallest)
  • I can compare digits column by column to determine which number is larger or smaller
  • I can compare and order decimal numbers with up to five decimal places

The Detective Strategy

To order large numbers correctly, you must be a place value detective. Follow these three steps:

The Three Steps

  1. The Digit Count: Check how many digits are in the number. More digits always means a larger value.
  2. Left-to-Right: If two numbers have the same amount of digits, start at the far left and compare.
  3. The Tie-Breaker: Move one column to the right until you find a digit that is different.
Detective Example: Which is larger: 4,567,800 or 4,562,999?
1. Both have 7 digits.
2. Millions: Both are 4.
3. Hundred Thousands: Both are 5.
4. Ten Thousands: Both are 6.
5. Thousands: One has 7, the other has 2.
Result: 4,567,800 is larger because 7,000 > 2,000.

Specialising
Put these numbers in ascending order:

5,300,000   530,000   5,030,000   503,000   5,300   53,000,000

Now look at your sorted list. Every number uses only the digits 5, 3, and 0. What made this set tricky to order? What did you have to pay attention to?
Try making your own set of 5 numbers using only the digits 2, 4, and 0. Can you make a set that would be tricky for a friend to order?
A mathematician's response

The correct order is: 5,300, 503,000, 530,000, 5,030,000, 5,300,000, 53,000,000.

At first glance I thought 5,300,000 and 530,000 might be close together because they both start with 53. But then I checked: 5,300,000 is about 10 times bigger than 530,000. The digits looked similar but the number of digits was completely different.

I nearly got confused by 5,030,000 and 503,000 too. They use the same digits in the same order, but the zeros are in different positions, making one about 10 times bigger.

What I noticed: the number of digits matters more than which digits you see first. Two numbers can use exactly the same digits but be wildly different in size if they have different numbers of digits. The zeros are what make it tricky because they change how many digits there are without changing which non-zero digits you see.

Mathematician's thinking: You are specialising. That means you are trying specific examples to gather evidence, looking for a pattern before you make any claims.
Conjecturing
"When comparing two whole numbers, the number with more digits is always the bigger number."
Is this always true, sometimes true, or never true? Test it with at least 5 different pairs of numbers. Try to find a pair where it breaks.
A mathematician's response

I tested this with several pairs:

45 (2 digits) vs 8 (1 digit). 45 is bigger. More digits wins.

999 (3 digits) vs 1,000 (4 digits). 1,000 is bigger. More digits wins again.

50,000 (5 digits) vs 123,456 (6 digits). 123,456 is bigger. Still working.

I tried hard to find one that breaks. What about 100 (3 digits) vs 99 (2 digits)? 100 is still bigger. What about 1,000,000 vs 999,999? 1,000,000 has more digits and is bigger.

My conjecture: always true for whole numbers. The smallest number with, say, 4 digits is 1,000, and the biggest number with 3 digits is 999. So a 4-digit number is always at least 1,000, which is always bigger than any 3-digit number. This works for every number of digits.

Mathematician's thinking: You are conjecturing. That means you are making a claim that you believe is true based on your evidence, but you know it still needs to be proved.
Justifying
A student put these numbers in ascending order:

34,500   340,500   345,000   3,450,000   3,405,000

Their teacher said one pair of numbers is in the wrong order. Which pair, and why?
Find the mistake, explain exactly what the student got confused about, and write the correct order.
A mathematician's response

First I checked each number's value:

34,500 = thirty-four thousand five hundred. That's the smallest, so good start.

340,500 = three hundred and forty thousand five hundred. Next I see 345,000 = three hundred and forty-five thousand. Is 340,500 smaller than 345,000? Yes, because both are in the hundred-thousands but 340,500 < 345,000. So far so good.

Now the last two: 3,450,000 and 3,405,000. Both are 7-digit numbers starting with 3. The student put 3,450,000 before 3,405,000. But 3,405,000 is less than 3,450,000 because in the ten-thousands column, 0 is less than 5.

The student probably looked at the digits 3,4,5 in 3,450,000 and 3,4,0,5 in 3,405,000 and thought the 5 appearing earlier made it smaller. But position matters: you have to compare column by column from left to right.

Correct order: 34,500, 340,500, 345,000, 3,405,000, 3,450,000.

Mathematician's thinking: You are justifying. That means you are not just checking whether an answer is right. You are examining the reasoning behind a mistake and explaining exactly where it went wrong.
