Learning Intention and Success Criteria

Learning Intention

We are learning to arrange numbers in ascending and descending order using place value reasoning.

Success Criteria

  • I can explain what ascending and descending order mean.
  • I can compare two numbers by reading their digits from left to right.
  • I can arrange a set of numbers from smallest to largest (ascending).
  • I can arrange a set of numbers from largest to smallest (descending).

Can You Sort These?

4,521 | 4,512 | 4,251 | 4,152 | 4,125
Key Idea

These numbers all start with 4. How do you decide the order? You'll need to look carefully at every digit.

Ascending order: 4,125 < 4,152 < 4,251 < 4,512 < 4,521

When the leading digits match, move to the next digit to the right and compare.

Card 3 of 10

The Left-to-Right Strategy

Compare 5,451 and 5,441:

Step 1Start from the left. Both have 5 in the thousands. Same, so move right.
Step 2Both have 4 in the hundreds. Same again, so move right.
Step 3Now the tens: one has 5, the other has 4. 5 is bigger than 4, so 5,451 > 5,441.
Step 4We didn't even need to look at the ones digit.
Key Idea

Compare digits from left to right. As soon as one digit is bigger, that number is bigger. You can stop there.

Card 4 of 10

What If Numbers Have Different Lengths?

Compare 987,654 and 87,654:

Step 1987,654 has six digits. 87,654 has five digits.
Step 2A number with more digits is always bigger (when we're talking about whole numbers).
Step 3So 987,654 > 87,654. We don't even need to compare individual digits.
Key Idea

Always check the number of digits first. More digits means a bigger number.

What strategies help when ordering gets tricky?

Investigation thread: What strategies help when ordering gets tricky?

Task 1: Specialising
Using only the digits 4, 4, 7, 2, 0, and 9 (each used exactly once), find the largest possible 6-digit number and the smallest possible 6-digit number. What is the difference between them?
Remember: a number cannot start with zero.
Model Response

For the largest, I put digits in descending order: 9, 7, 4, 4, 2, 0 → 974,420.

For the smallest, I need the smallest non-zero digit first: 2. Then remaining in ascending order: 0, 4, 4, 7, 9 → 204,479.

Difference: 974,420 - 204,479 = 769,941.

I initially tried putting 0 first for smallest but 044,279 isn't a 6-digit number.

Specialising
Task 2: Conjecturing
"If two numbers have the same number of digits and the same first digit, you always need to check every remaining digit to put them in order." Is this always true, sometimes true, or never true?
Try pairs of numbers where you can stop early, and pairs where you need to check every digit.
Model Response

Test 1: 54,321 and 59,876. Same first digit (5). Second digit: 4 vs 9. I can stop there. 59,876 is bigger. I did NOT need every digit.

Test 2: 55,431 and 55,439. Same first two digits. Third: 4 vs 4. Fourth: 3 vs 3. Fifth: 1 vs 9. I needed to check all five digits.

Test 3: 12,345 and 12,346. Same until the last digit. Needed all five.

So it's SOMETIMES TRUE. You only need to check every digit when all the leading digits match. Often you can stop early.

Conjecturing
Task 3: Generalising
Write a set of rules (like a recipe) that someone could follow to put ANY set of numbers in ascending order. Your rules should work for numbers of any size.
Test your rules with a tricky set like: 5,000,000 | 499,999 | 5,000,001 | 50,000 | 5,100,000
Model Response

My first attempt at rules: (1) Group numbers by digit count. Fewer digits = smaller. (2) Within same digit count, compare from left to right. (3) At each position, the number with the smaller digit goes first. (4) If digits match, move to the next position. (5) If all digits match, the numbers are equal.

Testing with the tricky set: 50,000 (5 digits), 499,999 (6 digits), 5,000,000 (7 digits), 5,000,001 (7 digits), 5,100,000 (7 digits).

Step 1: 50,000 is smallest (fewest digits). Step 2: 499,999 next (6 digits). Step 3: Three 7-digit numbers remain. Compare left to right: all start with 5. Second digit: 0, 0, 1. Two have 0, one has 1. So 5,100,000 is largest. Compare 5,000,000 and 5,000,001: identical until last digit. 0 < 1. So 5,000,000 < 5,000,001.

Final order: 50,000, 499,999, 5,000,000, 5,000,001, 5,100,000. My rules work!
