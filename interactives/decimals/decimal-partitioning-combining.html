Learning Intention

We are learning to practise partitioning and combining decimals, including recognising when a zero placeholder is needed.

Success Criteria

  • I can break a decimal into its place value parts
  • I can combine parts like 2 tenths + 3 thousandths and write the correct decimal (0.203)
  • I can explain why zero placeholders are needed in some decimal numbers

How Decimal Partitioning Works

Just like whole numbers live in "houses", decimal digits each have a place value. You can break a decimal into parts, and sometimes the parts don't look like you'd expect!

Standard vs Non-Standard Forms

  • Standard: 0.123 = 0.1 + 0.02 + 0.003 (each digit gets its own place)
  • Non-standard: 0.123 = 1 tenth + 23 thousandths
  • Non-standard: 0.123 = 12 hundredths + 3 thousandths
  • Zero trap: 2 tenths + 3 thousandths = 0.203 (the hundredths place needs a zero!)
Example "2 tenths + 3 thousandths" → the hundredths house is empty, so we fill it with 0 → 0.203

Decimal Partitioning Investigations

You've completed all three levels! Now it's time to explore decimals in deeper, more creative ways. These challenges will help you discover patterns and connections in how decimals work.

Task 1: Specialising (What if...?)
The decimal 0.75 can be partitioned as 0.7 + 0.05. But what if you could only use hundredths? Then it would be 0.01 + 0.01 + 0.01 + ... (75 times). That's not very useful! But what about using ONLY tenths and hundredths in different combinations? How many different ways can you partition 0.75 into exactly two parts, where each part is either a number of tenths or a number of hundredths?
List all the ways you find. Is there a pattern?
A mathematician's response

Let me try. 0.75 = 0.7 + 0.05 (7 tenths + 5 hundredths). What about 0.6 + 0.15? That's 6 tenths + 15 hundredths. 0.5 + 0.25? 5 tenths + 25 hundredths. 0.4 + 0.35, 0.3 + 0.45, 0.2 + 0.55, 0.1 + 0.65. And 0.0 + 0.75 if we allow zero tenths.

I initially thought there might only be a few ways, but the pattern is clear: every time I take away one tenth (0.1), I need to add ten hundredths (0.10) to keep the total at 0.75. That gives us 8 ways (from 0 tenths to 7 tenths). This is because 1 tenth = 10 hundredths.

Mathematician's thinking: You are specialising. That means you are trying specific examples to gather evidence, looking for a pattern before you make any claims.
Task 2: Conjecturing (Always/Sometimes/Never)
When you partition a decimal into its place value parts, the tenths part is always bigger than the hundredths part.
Is this always true, sometimes true, or never true? Find examples to support your answer.
A mathematician's response

Let me check some examples. In 3.84, tenths = 0.8 and hundredths = 0.04, so tenths IS bigger. In 3.48, tenths = 0.4 and hundredths = 0.08, tenths is still bigger.

Wait, let me think more carefully. Even if the hundredths digit is 9 (like in 3.19), the hundredths part is only 0.09, while the tenths part is 0.1. Since 0.1 > 0.09, the tenths part is always... actually, what about 3.05? The tenths part is 0.0 and the hundredths part is 0.05. So 0 < 0.05. That breaks it!

Answer: SOMETIMES. It's true unless the tenths digit is 0.

Mathematician's thinking: You are conjecturing. That means you are testing a claim to see if it is always true, sometimes true, or never true. Finding even one counterexample is enough to prove an "always" statement false.
Task 3: Justifying (Connection Task)
A student says: "Partitioning 4.56 and partitioning 456 are completely different things because one has a decimal point and one doesn't." Explain how the two processes are actually connected. What is the same about them, and what is different?
Think about place value in both cases.
A mathematician's response

At first I thought the student had a point, because decimals feel different. But let me look closely. 456 = 400 + 50 + 6. Each part matches a column: hundreds, tens, ones. 4.56 = 4 + 0.5 + 0.06. Each part matches a column: ones, tenths, hundredths.

The PROCESS is identical: break the number into its place value columns. The difference is just which columns you use. In both cases, each column is 10 times bigger than the one to its right. 400 is to 50 as 4 is to 0.5, they're both "10 times" relationships. The decimal point doesn't change how place value works. It just tells you where the ones column is.

Mathematician's thinking: You are justifying. That means you are building a clear argument that explains why something is true, not just that it is true. You connected two ideas (whole number partitioning and decimal partitioning) to show they follow the same structure.

Great work exploring these deeper concepts!
