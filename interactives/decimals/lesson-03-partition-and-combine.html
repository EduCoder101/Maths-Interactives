Breaking 0.123 Into Pieces

Meet our number: 0.123

0.123
Ones 0 0
Point .  
Tenths 1 0.1
Hundredths 2 0.02
Thousandths 3 0.003

Each digit lives in its own "house" (place value column). To expand, we write the value of each digit separately.

2
Standard Expanded Form

Each digit on its own

In standard expanded form, every digit keeps its own place value column, just like writing the address for each house individually.

Standard Expanded Form
0.123 = 0.1 + 0.02 + 0.003
1 tenth + 2 hundredths + 3 thousandths
✅ How It Works

Take each digit, multiply it by its column value, and add them all together:
(1 × 0.1) + (2 × 0.01) + (3 × 0.001) = 0.1 + 0.02 + 0.003 = 0.123

3
Non-Standard Expanded Form

Grouping digits together

In non-standard form, we can combine neighbouring digits into groups. There are several ways to do this!

Non-Standard Form A
0.123 = 0.1 + 0.023
1 tenth + 23 thousandths
Non-Standard Form B
0.123 = 0.12 + 0.003
12 hundredths + 3 thousandths
Non-Standard Form C
0.123 = 0.123
123 thousandths
🔑 Key Insight

Non-standard doesn't mean "wrong", it simply means the digits are grouped differently. All of these forms are equal to 0.123. The standard form just keeps each digit in its own column.

4
⭐ The Rule

Standard expanded form = each digit in its own place value column.
Non-standard form = digits grouped across columns. Both must always add up to the same original number.

5

🤔 Why learn non-standard forms?

Non-standard forms help with mental strategies. For example, when adding 0.123 + 0.077, it's easier to think: "123 thousandths + 77 thousandths = 200 thousandths = 0.2". You're grouping all the parts into the same unit to make the addition simpler!

Being flexible with how you partition decimals is a key skill for addition, subtraction, and problem solving.

Go Deeper Extension
Partition & Combine Investigations

These tasks push your thinking beyond just partitioning and combining. You will investigate which decimals have more ways to be partitioned, and why the rules always work.

Task 1: Specialising
Do all decimals have the same number of non-standard forms? Try these three and count.
For each decimal, write as many different non-standard forms as you can find.

Try these:
A: 0.24 (hint: what about 24 hundredths as a single group?) B: 0.456 (this one has the most to explore) C: 0.3 (what about this one? Is it different?)

Mathematician's Thinking

Let me try each one:

0.24: Standard form: 0.2 + 0.04 (2 tenths + 4 hundredths).
Non-standard: 0.24 (24 hundredths). That's 2 forms total.

0.456: Standard form: 0.4 + 0.05 + 0.006.
Non-standard forms:
– 0.45 + 0.006 (45 hundredths + 6 thousandths)
– 0.4 + 0.056 (4 tenths + 56 thousandths)
– 0.456 (456 thousandths)
That's 4 forms total!

0.3: Standard form: 0.3 (just one tenths digit).
Non-standard: 0.3 again... wait, is that the same? With just one decimal place, there is only 1 way to write it.

What I notice: Decimals with more digits have more ways to partition them. A decimal with 3 decimal places (like 0.456) can be split into groups of neighbouring digits in 4 different ways!

Pattern: The more decimal places a number has, the more non-standard forms you can create. A 2-digit decimal has 2 forms. A 3-digit decimal can have up to 4 forms!
Task 2: Conjecturing
When you rewrite a non-standard form back into standard form, do you always get the same decimal?
Make a conjecture (a guess) about whether this is always true, sometimes true, or never true. Test it with an example.

Try this example:
Original non-standard form: 0.3 + 27 thousandths This equals: 0.3 + 0.027 = ? Now write 0.327 in standard form. Do you get back to 0.3 and 27 thousandths? What does this tell you about the reliability of partitioning?

Mathematician's Thinking

My conjecture: It always works. If I regroup and recombine, I will get the same decimal back.

Let me test with 0.3 + 27 thousandths:

0.3 = 0.300 (to see it with thousandths)
27 thousandths = 0.027
0.300 + 0.027 = 0.327

Now if I write 0.327 in standard form:
0.327 = 0.3 + 0.02 + 0.007

Hmm, that's 0.3 + 0.027 rewritten differently. I got back to the same decimal (0.327), but I didn't get back to the same non-standard grouping (27 thousandths). That's okay though!

Why this works: Every non-standard form must equal the same decimal, because we are just rearranging the same "pieces". The place value adds up the same way no matter how we group the parts.

Key insight: If you regroup a non-standard form correctly, you always get a true equation. The groups are just different arrangements of the same decimal. The math never lies!
Task 3: Justifying
A student made an error when partitioning a decimal. Can you find it and explain why it is wrong?
Identify the mistake and explain how to fix it.

Student's work:
"I partitioned 0.514 like this:
0.5 + 1 hundredth + 4 thousandths = 0.5 + 0.01 + 0.4"

The student wrote this as standard form: 0.5 + 0.01 + 0.4 = 0.91
But 0.514 is definitely not the same as 0.91!

Mathematician's Thinking

Where is the error? The student wrote "4 thousandths" but then wrote it as 0.4, which is 4 tenths, not 4 thousandths!

Let me check:
0.514 = 0.5 + 0.01 + 0.004 (standard form)

But the student wrote:
0.5 + 0.01 + 0.4 = 0.91

The mistake: They confused the place value. They said "4 thousandths" (which equals 0.004) but then used 0.4 (which is 4 tenths). These are completely different!

How to fix it: Write it carefully step by step:
0.514
= 5 tenths + 1 hundredth + 4 thousandths
= 0.5 + 0.01 + 0.004
= 0.514 ✓

Key learning: When you partition a decimal, you must be very careful about place values. Tenths, hundredths, and thousandths are not the same! Write the zeros carefully. 4 thousandths is 0.004, not 0.4.
