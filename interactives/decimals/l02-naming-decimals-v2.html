Learning Intention and Success Criteria

Learning Intention

We are learning to read and name decimal numbers using the 'last name' rule.

Success Criteria

  • I can read a decimal by looking at its last digit's place value.
  • I can name a decimal using the correct place value term (tenths, hundredths, or thousandths).
  • I can explain why 0.6 and 0.60 have the same value but different names.
  • I can write a decimal when given its spoken name.

Card 3 of 10

How Do We Read 0.321?

Step 1Read the digits after the decimal: 321
Step 2Find the last digit: 1. It is in the thousandths place.
Step 3So the last name is 'thousandths'.
Step 4Read it: three hundred and twenty-one thousandths
Remember

Always read the digits as a whole number, then add the last name.

Card 4 of 10

Is 0.6 the Same as 0.60?

Step 10.6: the last digit is in tenths = 'six tenths'
Step 20.60: the last digit (0) is in hundredths = 'sixty hundredths'
Step 3Are 6 tenths and 60 hundredths the same? Think of money: 6 ten-cent pieces = 60 cents. Yes, same value!
Step 4Both equal 0.6. The trailing zero doesn't change the value, but it changes the name.
Key Idea

0.6 = 0.60 = 0.600. Same value, different names. The 'last name' depends on how many decimal places are written.

Go Deeper: Naming Decimals Investigations

These tasks will push your thinking beyond finding answers. You will explore, make conjectures, and reason like a mathematician.

Specialising
Write three different decimals that ALL equal 0.5 but have different names. For each one, write both the decimal and its name.
Think about trailing zeros.
A mathematician's response

0.5 = "five tenths"

0.50 = "fifty hundredths"

0.500 = "five hundred thousandths"

All equal 0.5 but have different names because the last digit sits in a different place value column each time. I could keep going: 0.5000 = "five thousand ten-thousandths". The value never changes, but the name keeps changing.

Key insight: Trailing zeros are invisible to the value but visible to the name.
Conjecturing
'A decimal with more digits after the decimal point is always a larger number.' Is this always, sometimes, or never true?
Compare 0.9 (1 digit) with 0.123 (3 digits). Then compare 0.1 (1 digit) with 0.99 (2 digits).
A mathematician's response

Let me compare: 0.9 vs 0.123

0.9 = "nine tenths" = 900 thousandths
0.123 = 123 thousandths
So 0.9 is bigger even though it has fewer digits!

Now 0.1 vs 0.99:
0.99 is bigger AND has more digits.

And 0.5 vs 0.50: same value, different digit count.

The answer is SOMETIMES TRUE. The number of digits tells you nothing about size. You must compare the actual values.

Key insight: More digits does not mean a bigger decimal. Always compare the tenths place first, then hundredths, etc.
Generalising
Write a rule for converting between names. How many hundredths are in 1 tenth? How many thousandths are in 1 hundredth?
Can you see a pattern?
A mathematician's response

Let me work this out:

1 tenth = 10 hundredths (because 0.1 = 0.10)
1 hundredth = 10 thousandths (because 0.01 = 0.010)
1 tenth = 100 thousandths (because 0.1 = 0.100)

I notice a pattern: each time I move one place value to the right, I multiply by 10.
So: 3 tenths = 30 hundredths = 300 thousandths

My rule: to convert to a smaller place value, multiply by 10 for each step right.

Key insight: Place value for decimals works with the same powers of 10 as whole numbers, just in the opposite direction.
