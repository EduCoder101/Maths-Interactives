Learning Intention and Success Criteria

Learning Intention

We are learning to partition decimals into standard expanded form and non-standard combinations.

Success Criteria

  • I can write a decimal in expanded notation using tenths, hundredths, and thousandths.
  • I can build a decimal from given parts.
  • I can partition a decimal in a non-standard way.
  • I can explain why a zero placeholder is needed in some partitions.

Numbers Are Built from Parts

Just like LEGO, this number is built from smaller pieces.

0.123

Snap them together:

0.1 (blue block, tenths)
0.02 (purple block, hundredths)
0.003 (red block, thousandths)

0.1 + 0.02 + 0.003 = 0.123

Card 2 of 10

Can We Build It Differently?

The same number can be broken apart in more than one way:

Standard way: 0.1 + 0.02 + 0.003 = 0.123
Non-standard way 1: 0.12 + 0.003 = 0.123 (12 hundredths + 3 thousandths)
Non-standard way 2: 0.1 + 0.023 = 0.123 (1 tenth + 23 thousandths)
Big Idea

There are many ways to break a number apart. Standard expanded form splits every digit. Non-standard lets you group digits differently.

Card 4 of 10

2 tenths + 3 thousandths = ?

Step 12 tenths = 0.2
Step 23 thousandths = 0.003
Step 3What goes in the hundredths place? Nothing! We need a zero placeholder.
Step 4The answer is 0.203. The zero holds the hundredths place.
Key Idea

When a place value has no part assigned to it, you must fill it with zero.

Specialising
How many different ways can you partition 0.5 into exactly two parts where both parts are decimals with up to 3 decimal places? List at least 5 different ways.
Can you find a partition where one part uses thousandths?
A mathematician's response

0.5 = 0.4 + 0.1. Also 0.5 = 0.3 + 0.2. 0.5 = 0.25 + 0.25. 0.5 = 0.499 + 0.001. 0.5 = 0.350 + 0.150. Yes, one part uses thousandths: 0.5 = 0.499 + 0.001. There are actually infinitely many if we allow any combination of decimals. If we just look at pairs of tenths: (0.1, 0.4), (0.2, 0.3). Pairs of hundredths add another 49 possibilities. Pairs of thousandths add hundreds more. The more decimal places we allow, the more ways we can partition.

Mathematician's thinking: You are specialising. That means you are trying specific examples to gather evidence, looking for a pattern before you make any claims.
Conjecturing
"A decimal in expanded form always has the same number of terms as it has non-zero digits."
Always, sometimes, or never true? Test with 0.456, 0.400, and 0.050.
A mathematician's response

0.456 has 3 non-zero digits and 3 terms: 0.4 + 0.05 + 0.006. 0.400 has 1 non-zero digit and 1 term: 0.4. 0.050 has 1 non-zero digit and 1 term: 0.05. Also test 0.101: 2 non-zero digits and 2 terms: 0.1 + 0.001. Seems ALWAYS TRUE. In standard expanded form, each non-zero digit creates exactly one term. Zero digits are skipped. So the number of terms always equals the number of non-zero digits after the decimal.

Mathematician's thinking: You are conjecturing. That means you are making a claim that you believe is true, based on your evidence, but you know it still needs to be proved.
Generalising
Write a rule explaining when you need a zero placeholder when combining parts into a decimal. When can you skip the zero?
Think about what happens when you add 3 tenths + 5 thousandths versus 3 tenths + 5 hundredths.
A mathematician's response

3 tenths + 5 hundredths: tenths = 3, hundredths = 5, no gap → 0.35. No placeholder needed. 3 tenths + 5 thousandths: tenths = 3, hundredths = ?, thousandths = 5. There's a gap at hundredths → need 0 → 0.305. My rule: you need a zero placeholder whenever there is a "gap" between the given parts. If parts are given for tenths and thousandths but not hundredths, the hundredths must be filled with zero. The zero ensures every digit sits in the right column.

Mathematician's thinking: You are generalising. That means you are looking for a broad rule or pattern that works across many different examples.
